금융ICT에서 DevOps, SRE를 이야기합니다.

I am Ryan Cho.

Site Reliability Engineer

 저는 devops, SRE에 관심이 많은 시스템 관리자 입니다. “코드로 관리하는 인프라”라는 말은 제가 일하고 있는 금융권에서는 아직 적용되지도, 배우기도 힘든 분야 입니다. 하지만 더 이상 미루다가는 제 개인적인 기술부채가 갚지 못할 정도로 커져 따라잡을 수 없을 것이란 불안감으로 이 블로그를 시작합니다.

Work

 신한은행의 인터넷뱅킹과 신한SOL App의 서버쪽 운영을 담당하고 있습니다. 주로 하는 일은 Site Reliability Engineering으로 365일 24시간 안정적인 서비스를 제공할 수 있도록 이곳, 저곳을 살피는 것 입니다. Web서버와 WAS, Business Logic을 담당하는 TP-Monitor MiddleWare를 운영하는 것이 제 일 입니다.

Family

 너무나 사랑스런 와이프와 사춘기가 오고 있는 귀여운 딸, 개구쟁이 아들이 있습니다.
“아빠는 오늘도 늦게와?”에서 아빠를 담당하고 있습니다.

Technical Stack

 ICT업계에서 10년 넘게 일하고 있는데 이렇게 기술 스택을 정리할 때면 한없이 부족해 보입니다. 한 때 “저걸 새로 배워서 몇년이나 쓴다고”라는 편협한 생각으로 공부를 미룬 때도 있었습니다. 아직 젊다는 생각으로 DevOps에 관련된 기술들을 배우고 있습니다. 프로그래밍 실력이 미천하여 여러 곳에서 넘어지고 있지만 한 걸음 한 걸음 앞으로 가고 있습니다.

  1. Legacy (이미 알고 있는 것)
    • System : Superdom (hpe Mission Critical System)
    • OS : Linux hp-ux AIX
    • Web Server : Apache NginX WebtoB
    • WAS : WebLogic JEUS
    • TP-Monitor : tmax tuxedo
    • APM : Jennifer Pharos
    • Language : Java (java 프로그래밍이라기 보다 JVM 성능 관리, GC 등 운영 관점의 java) Shell script
    • DB : Oracle (SQL 관점이 아니라, RAC 등 운영 관점의 Oracle DB 관리)
  2. Devops (새로 배우고 있는 것)
    • Language : python
    • Cloud : AWS Azure GCP
    • SCM : Git github gitlab
    • Container : Docker
    • Orchestration : Kubernetes (#어렵네요)
    • CI/CD : Jenkins
    • Automation : Ansible Terraform
    • APM : Datadog
    • DB : MySQL
